DataManager Usage Instructions

Installation

composer require fereydooni/data-manager

Configuration

If using Laravel, publish the config file:

php artisan vendor:publish --provider="DataManager\Providers\DataManagerServiceProvider" --tag=config
Tip: You can customize default format, chunk size, logging, and error handling in config/data-manager.php.

Core Features

  • Import and export data in multiple formats: CSV, JSON, XML, SQL, Excel, database models, Spatie Data objects
  • Custom importers/exporters via interfaces
  • Data validation and transformation hooks
  • Batch processing for large datasets
  • Logging, error handling, and progress tracking utilities
  • Event hooks for import/export lifecycle
  • Extensible and configurable

Utilities

  • Logger: DataManager\Utils\Logger::log('message');
  • Exception: throw new DataManager\Exceptions\DataManagerException('error');
  • Progress Tracker: 
    $tracker = new DataManager\Utils\ProgressTracker($total);
foreach ($items as $item) {
    // ...
    $tracker->advance();
}
$tracker->finish();

Extending

  • Implement ImporterInterface or ExporterInterface for custom formats.
  • Use DataManager\Utils\EventDispatcher for event hooks:
use DataManager\Utils\EventDispatcher;
EventDispatcher::listen('import.started', function ($source) {
    // ...
});
EventDispatcher::dispatch('import.started', $source);

Configuration Options

  • default_format: Default format for import/export (e.g., csv)
  • chunk_size: Batch size for processing
  • logging: Enable/disable logging
  • error_handling: Log errors, throw exceptions, etc.
